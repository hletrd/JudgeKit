## 오늘 새로 본 후보자 측 이슈

### 🟡 시험 중 채점 시스템 장애 시 후보자 측 알림 부재 (High, 오늘 사고의 후보자 경험)
- 14h 동안 모든 제출이 compile_error로 잘못 채점되는 상태에서 후보자가 응시했다면, 본인 코드는 멀쩡한데 "compile_error" 받음
- 시간 제한 안에 다른 언어로 시도해도 동일 결과
- 응시 시간 다 쓰고 결과 0점
- 후보자 입장에선 본인 잘못으로 보임 → 추후 분쟁
- **수정**: 채점 시스템 헬스 모니터링 + 후보자 화면에 "현재 채점 시스템 점검 중" 배너 + 자동 시간 연장.

### 🟡 모바일 ARM64 후보자 환경에서 일부 언어 미지원 자각 부재 (Low)
- moonbit, odin 같은 niche 언어는 amd64 emulation 필요
- 후보자가 그 언어로 시도하면 채점 시간 길어짐
- 활성 언어 카탈로그에 "이 언어는 채점이 느릴 수 있어요" 표시 부재

### 🟡 글로벌 채용 시 데이터 보호 알림 부족 (Med, 어제 컴플라이언스 갭과 연결)
- 응시 시작 동의 화면이 anti-cheat 모니터링을 명시는 함
- 다만 데이터 보존 기간·국외 이전·삭제 권리 명시 부족
- GDPR/PIPA 적용 지역 후보자에게 응시 전 명시 권장

## Show-stopper 후보 (어제 동일)

- **시스템 체크 페이지 + 서버 드래프트 복원** — 채용 운영 시 후보자 사고 케이스 회피용.

## 추천 작업 순서

1. 시험 전 시스템 체크 페이지 (5분 사전 검증)
2. 서버 측 드래프트 복원
3. 채점 시스템 헬스 → 후보자 화면 배너 (오늘 사고 후속)
4. Heartbeat grace 정책
5. 후보자 계정 마감 후 처리
6. 모바일 안내 또는 미지원 명시
7. (장기) 신분증 업로드, 웹캠 keepalive
